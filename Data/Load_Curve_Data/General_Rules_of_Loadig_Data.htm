General Rules of Loading Data

GEO can load several different types of ASCII files, such as CWLAS (LAS), tab-delimited, space-delimited and comma-delimited.

If the ASCII file is not a LAS file, there are some basic requirements to follow prior to loading.

  • The curve data must be laid out in rows and columns.

  • The first row of the file must contain the curve names or curve mnemonics (such as ROP, GR), and must not be blank.

  • The first column must always be depth.

  • Each curve mnemonic or unit must not contain any spaces and must not exceed thirty-two characters.

  • The last row of the file must be the last row of the data set.

  • The delimiter used at the first column must be same for rest for data. Thus, you cannot have a data file that is delimited by comma and tab.

  • As an option, the second row of the file may be the curve unit (such as ft/hr, %, or GAPI).

Deviation from the above may cause errors during loading data from files. When such errors occur, it is advisable to examine the data set to determine the cause of the problem.

Tip. If possible, ask for LAS files from your data provider. LAS format is the easiest curve data format to load. If the vendor cannot provide LAS file, please ask them to give you the Tab-delimited ASCII file.
