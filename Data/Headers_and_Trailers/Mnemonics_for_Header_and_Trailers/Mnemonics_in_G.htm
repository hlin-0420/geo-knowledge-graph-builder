Mnemonics in GEO

In a plot header or trailer, you need to be able to fill in well or log information such as well name, field name, company name, vendors name, name of the geologists, coring intervals, casing intervals etc.

 

To do this, a specific mnemonic is assigned to each piece of information in the plot header or trailer. All mnemonics must be unique within the ODF file. The software uses the mnemonic ID to uniquely identify a mnemonic.

 

A mnemonic consists of the following;

Component

Example

Mnemonic Identifier (ID)

WD

Description

Well Depth

Value

115

Units

FT

View in Shell

No or Yes (Yes enables the mnemonic value to display when selecting the .odf file in windows explorer)

 

 

Add mnemonic to header

The following steps demonstrate how to add a Field Name mnemonic to the header. If this mnemonic is added to the header of other layouts in the odf, then any changes made to the mnemonic Value will be changed in the other layout headers also.

 

Please note that mnemonics can only be added to headers if your license includes GEOVectdraw.

 

  1. Open the Header in GEOVectdraw. 
  2. Use the Mnem Desc/Val/Unit tool to add a mnemonic. The Mnemonic (Group) Properties dialog box will open.
  3. Enter the following information;
    1. Mnemonic - FN
    2. Mnemonic Description - Field Name
    3. Mnemonic Value - Drent
    4. Note. Leave System Variable clear
  4. Save your changes in vectdraw.

 

Once the header has been finalized, you will see Field Name: Drent in each ODF header:

 

 

Edit mnemonic

Continuing the example above, the 'Field Name' (Mnemonic Description) and 'Drent' (Mnemonic Value) can be edited in GEO using one of these two methods;

  • Edit directly in the ODF by left clicking the mnemonic description or value in the plot header or trailer.

  • From the Define Mnemonics dialog box - on the Define tab, select Mnemonics . Here, you can also define pre-set values that can be selected from a drop list in the header.

Note. GEOVectDraw is not be required for these alterations. However, any changes made via these methods are only valid/happen in that particular ODF.

 

 

Add mnemonic values from Las Files : Information Blocks

When a LAS file is loaded to the GEO Software Suite, the software reads in and stores all the available mnemonics in the Parameter Information block and the Well Information block from that particular LAS file. GEO then maintains these mnemonics on a per file basis. The data in these blocks forms the basis for the final values for each mnemonic in a particular log.

 

If data is loaded in a non-Las format, but you wish to export it from GEO into Las format, then mnemonics will not automatically have their information block assigned. However, this can be done manually in GEO.

  1. On the Define tab, select Mnemonics. The Define Mnemonics will open.

  2. All data files loaded into the ODF will be available in the File drop down list at the top. The corresponding mnemonics from those files will be listed in the box directly below in the mnemonics area.

  3. To add a mnemonic to the plot,

    1. Multi-select by dragging the mouse over a range of entries in the mnemonics area, or hold Ctrl and click the individual mnemonics you wish to add. They should now have a blue highlight

    2. Click Add to Plot. The appropriate header will now populate with the information provided the mnemonics in the header VOB match.

    3. Click Exit to return to the plot area.

  4. In the Plot Mnemonics section to the right of the table you will see a column labeled Information Block.

  5. Enter the letters P, W or O to stipulate which block you wish to assign the mnemonic to, Parameters, Well or Other.

  6. Do this for each mnemonic you wish to appear in the required block and click Exit.

 

If you now wish to export a curve as XML, the mnemonics assigned as Parameter Information will now populate in the file.

 

See the section on loading XML files to learn more about importing XML data.

 

Tips. Using the API recommended mnemonic (while designing the header with GEOVectDraw) will help the user to complete the header information, whenever a LAS file is loaded into the software.

Select the corresponding LAS file (from the File list box in Define Mnemonics dialog box) to determine the appropriate mnemonic values for your plot header.

The possible length of a mnemonic is 32 characters. The API standards have not changed, so most mnemonics will still be three or four characters, but you have the option of creating user defined mnemonics that are longer.
