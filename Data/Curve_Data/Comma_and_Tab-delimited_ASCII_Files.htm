Comma and Tab-delimited ASCII Files

You can load data files or either comma-separated (CSV) or tab-delimited text type without additional manipulated via the Data Import Wizard. 

In a comma-delimited ASCII file, cells in a column are separated from the next by a comma, while in a tab-delimited ASCII file, cells in a column are separated from the next by a comma, while in a tab-delimited ASCII file, cells are separated by a tab character.

In a comma-delimited ASCII file, cells in a column are separated from the next by a comma, while in a tab-delimited ASCII file, cells are separated by a tab character.

In those kinds of files, the range between commas or tabs is taken as the column width.

The application simply reads values between commas (or tabs) and ignores any spaces.
Compared to space-delimited data, these are more structured formats, and as such (together with the wholly structured LAS format, or compatible-XML format) are the preferred data formats to request from your vendors.
