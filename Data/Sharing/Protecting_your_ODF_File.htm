Protecting your ODF File

ODF files can be password protected should the content of the file be of a confidential nature, and/or you want to prevent certain recipients of the file from modifying or updating its content.

 

Use the following procedure to password your file.

  1. On the Define tab, select Passwords. The Specify Access for Document will open.

  2. From the Select Desired Access list, select the Update access level.

  3. In the Enter Password for Access, type the chosen password.

  4.  Re-confirm the password in the Re-enter Password box.
    Note. The password is not case sensitive and its maximum length is 15 characters.

  5. Click Update.

  6. Repeat this process for the View / Print and Export access level listed in the Select desired Access box.  

  7. Click Update.

 

Note. Remember to keep a record of your chosen password(s) in a secure place, as once password protected you will not be able to open the ODF, or change/remove password protection without the correct Update level password.

 

Next time the ODF file is saved, the password protection will be implemented at the chosen access level.

 

Each time the ODF file is opened thereafter, the appropriate password must be entered for the required access level.

 

Notes. You cannot apply a password for View / Print access, without first applying a password for Update access.

Password protection can be removed/changed only by opening the ODF with the Update access level password, and then deleting the entries in the password fields. You must delete the View / Print access password first, before attempting to remove the password for Update access.

 

Tip. If a password for Update access has been applied to an ODF, you basically send it as a ‘Read-Only’ file to another party (or simply like a picture). The other party will not have any access to the data that builds this ODF, unless they are separately provided with the appropriate password.
